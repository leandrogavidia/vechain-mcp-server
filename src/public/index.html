The server exposes one documentation tool and four Thorest API tools.

Vechain Docs: searchDocumentation

Accounts: getAccount

Server code notes

The server is built with @modelcontextprotocol/sdk over stdio. Input is validated with zod. Timeouts are enforced with AbortController and configurable via vechainConfig.mainnet.controllerAbortTimeout. Errors consistently return JSON with error and reason. 

import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";
import { vechainConfig } from "./config.js";
import { REVISION } from "./types.js";
import { createVechainDocsMcpClient } from "./client.js"; 

const server = new McpServer(
  { name: vechainConfig.mcpServer.name, version: vechainConfig.mcpServer.version },
  { capabilities: { tools: {} } },
);

// ... register tools: searchDocumentation, getAccount, getTransaction, getBlock, getPriorityFee

async function main(){
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error("Vechain MCP Server running on stdio");
}

main().catch((e)=>{ console.error("Fatal error in main():", e); process.exit(1);});

Configuration

Timeouts

Each tool uses an AbortController with a configurable timeout:

vechainConfig.mainnet.controllerAbortTimeout // ms

Validation

Addresses and tx IDs are validated with regex. All inputs pass through zod schemas. 

/^(0x)?[0-9a-fA-F]{40}$/ // address
/^0x[0-9a-fA-F]{64}$/   // tx id

Mainnet base URL

Point to a Thorest‚Äëcompatible endpoint:

vechainConfig.mainnet.thorestApiBaseUrl

FAQ

How do I query a specific block state? Use the revision parameter across tools that support it. For example, pass a block number to getAccount or getBlock to pin reads to that state.
Why am I seeing timeouts? Increase vechainConfig.mainnet.controllerAbortTimeout and verify your thorestApiBaseUrl is reachable from the host environment.
Can I return raw encodings? Yes‚Äîset raw: true on getTransaction or getBlock to request raw hex / RLP where supported.
