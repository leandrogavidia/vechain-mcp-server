This page documents and helps you configure an MCP server exposing VeChain Docs search plus Thorest API tools for accounts, transactions, blocks and fee suggestions.

Integrations

Register the server in your MCP‑aware host configuration (paths vary by host). 

{
  "mcpServers": {
    "vechain": {
      "command": "node",
      "args": ["server.js"],
      "env": { }
    }
  }
}

Tool playground

Build a request and copy the generated command or JSON arguments into your MCP host. This playground does not call the network; it simply formats requests for you.

🔎 searchDocumentation 
{
  "name": "searchDocumentation",
  "arguments": { "query": "" }
}
👤 getAccount 
{
  "name": "getAccount",
  "arguments": { "address": "" }
}
🔗 getTransaction 
{
  "name": "getTransaction",
  "arguments": { "id": "" }
}
📦 getBlock 
{
  "name": "getBlock",
  "arguments": { "revision": "best" }
}
⛽ getPriorityFee

FAQ

How do I query a specific block state? Use the revision parameter across tools that support it. For example, pass a block number to getAccount or getBlock to pin reads to that state.
Why am I seeing timeouts? Increase vechainConfig.mainnet.controllerAbortTimeout and verify your thorestApiBaseUrl is reachable from the host environment.
Can I return raw encodings? Yes—set raw: true on getTransaction or getBlock to request raw hex / RLP where supported.
